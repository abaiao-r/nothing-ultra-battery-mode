# Contributing

## Branch and PR Rules

1. Open a GitHub issue first.
2. Create a branch from `main`:
   - `feature/<issueid>-<ticket_name>`
   - `bugfix/<issueid>-<ticket_name>`
3. Keep one commit in the branch.
4. Open one PR per issue.
5. Fill all PR template sections.

## Merge Rules

- No direct push to `main`
- Required checks must pass
- Squash merge only (enable in repo settings)

## Required PR Checks

- PR Governance
- Android CI
